Popup Address-Detection Test Suite

25 community / neighborhood pages built to stress every common way real estate sites embed (or hide) a location.

How to read this matrix
  • URL hint — how much information is leaked by the URL alone (full address, city+state, just an ID, etc.).
  • Embed technique — where the actual address lives on the rendered page.
  • Expected output — what the popup should resolve to (a precise street address when possible, otherwise the neighborhood + city + state).
  • Difficulty — how likely a naive DOM scrape will find the address.
  • Each row mimics a real-world platform whenever possible (Redfin, Trulia, Realtor.com, Compass, Sotheby’s, Niche, Lennar, DR Horton, Toll Brothers, Cloudflare-protected sites, HOA portals, etc.).

URL pattern coverage

The 25 test pages span the following URL-hint categories:

  • Numeric ID only (worst case — no hint): 3 pages (community-15, community-id-8421, nh-32)
  • Hood only, no city/state: 4 pages (brookline-village, areas-georgetown, the-quarter, the-village-at-bishops-gate)
  • City + state, no hood: 2 pages (homes-in-sacramento-ca, lake-bluff-il-condos)
  • Hood + city + state in URL: 8 pages (Redfin/Trulia/Compass/Niche/Realtor.com/Lennar/Toll-Brothers/Walkscore styles)
  • City + state + zip: 1 page (scottsdale-az-85255)
  • Misleading / decoy URL: 2 pages (find-homes, coming-soon-build)
  • Generic content URL: 2 pages (our-favorite-neighborhoods, canyon-creek-hoa)
  • Agent or brokerage profile: 3 pages (agent-jane-doe-detroit, cb-clayton-mo-st-louis, cary-nc-55plus)

Embed technique coverage

The 25 pages exercise these distinct ways an address may live on a page:

  1. Visible H1 + breadcrumb (easy baseline)
  2. Plain semantic <address> element
  3. hCard microformat (class="adr")
  4. JSON-LD RealEstateAgent → worksFor.address
  5. Inline <script type="application/json"> data blob
  6. SPA window.__INITIAL_STATE__ in inline script
  7. <meta name="geo.placename"> / geo.region / geo.position / ICBM
  8. <meta name="twitter:title"> only
  9. Embedded Google Maps <iframe src>
  10. Google Maps "Get Directions" <a href>
  11. SVG <text> element label
  12. Cloudflare data-cfemail XOR-obfuscation
  13. HTML comment
  14. <noscript> block fallback
  15. <template> element (DOM but not rendered)
  16. Footer-only
  17. Image src URL filename
  18. PDF download link filename
  19. Multiple decoy addresses; primary visually highlighted
  20. Composed from URL slug + page content (no full address anywhere)
  21. City inferable only from indirect cues (subway lines, neighborhood names)
  22. Lat/long coordinates only (reverse-geocoding needed)
  23. Disambiguation of common name (Georgetown DC vs TX vs KY)
  24. Disambiguation of ambiguous abbreviation ("LA" = Los Angeles)
  25. Office address vs. neighborhoods-served list
